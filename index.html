Panduan Penggunaan HTTP Request

  • GET: Mengambil data dari server.
  • POST: Mengirim data baru ke server.
  • PUT: Mengganti data yang ada secara keseluruhan.
  • DELETE: Menghapus data dari server.
  • HEAD: Mengambil header respons tanpa body.
  • OPTIONS: Mengambil opsi komunikasi yang didukung server.
  • PATCH: Mengupdate sebagian data yang ada.
  • TRACE: Melacak rute request yang dikirim.
  • CONNECT: Menghubungkan ke server melalui proxy.
  • PROPFIND: Mengambil properti dari resource (umumnya WebDAV).

Pastikan URL valid (gunakan http:// atau https://) dan payload sesuai dengan method yang dipilih.


Contoh payload untuk request POST:

{
  "action": "create",
  "data": {
    "name": "RenXploit Tools",
    "type": "security",
    "status": "active"
  }
}
